    Nous avons modifié la table Personne pour y intégrer un objet JSON permettant d'entrer l'adresse. C'est utile dans la mesure où une adresse est très disparate et comprend beaucoup de champs différents. À la place de quatre entrées (numero, rue, ville, code_p), nous n'en avons plus qu'une : adresse.

    Nous avons également modifié le fonctionnement d'un dossier médical. Nous avions en SQL quatre tables reliées à la table Animal, représentant les quatre entrées possibles dans un dossier médical. Nous les avons remplacées par une unique table Dossier_medical qui contient un objet JSON dans lequel nous pouvons entrer toutes les informations nécessaires. Il y a également toujours une clé étrangère permettant de relier ce dossier médical à l'animal concerné. Nous avons réfléchi ici à la possibilité de placer cet objet JSON directement dans la table Animal mais cela ne permettrait plus de conserver les anciennes mesures.
    Lorsque nous aurons besoin de récupérer les données, il suffira de regarder à quelle date a été réalisée telle ou telle mesure, pour retrouver la plus récente et afficher les informations.
